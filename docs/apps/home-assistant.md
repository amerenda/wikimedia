# Home Assistant

Self-hosted home automation on Mac Mini M4. Configuration is fully managed as code in `mac-mini-compose`.

- **URL:** `ha.amer.dev`
- **Version:** 2026.3.4
- **Platform:** Docker Compose on Mac Mini (OrbStack), `host` network mode

## Why Mac Mini (Not k3s)

- **Host networking required** — mDNS/SSDP device discovery doesn't work through k8s NAT
- **Voice pipeline** — Whisper/Piper use Apple Silicon Metal GPU; in-cluster containers lose GPU access

A k3s config skeleton exists in `k3s-dean-gitops/apps/home-assistant/` for a future migration.

## Integrations

| Integration | Details |
|---|---|
| Philips Hue | Native integration; scenes auto-discovered (`scene.<room>_<window>`) |
| Zigbee (ZHA) | USB stick on rpi5-1 via ser2net at `socket://rpi5-1.amer.home:3333` |
| Reolink cameras | Motion detection automations (human/pet detect, privacy mode) |
| Voice pipeline | Whisper (STT) + Piper (TTS) + OpenWakeWord — see below |
| Tailscale | Remote access |
| PostgreSQL | History/statistics (unified Mac Mini Postgres) |

## Voice Pipeline

Three services run alongside HA on the Mac Mini:

| Service | Port | Purpose | Config |
|---|---|---|---|
| Whisper | 10300 | Speech-to-text | model: `tiny-int8`, lang: `en` |
| Piper | 10200 | Text-to-speech | voice: `en_US-lessac-medium` |
| OpenWakeWord | 10400 | Wake word detection | model: `ok_nabu` |

## Smart Lighting v2

Fully code-managed lighting automation. All config in `homeassistant/configuration/` — no manual HA UI editing needed.

**House modes:** Home / Away (lights off) / Sleep (motion-limited) / Guest

**Time windows:** morning / day / evening / night — weekday and weekend schedules differ

**Per-room behavior:**
- Motion triggers the scene for the current time window + house mode
- Manual override clears at the next time window boundary
- Input helpers (booleans, datetimes, selects, numbers) control all parameters
- Room selector dashboard for per-room config

Scene naming convention: `scene.<room>_<window>` (e.g., `scene.living_room_evening`) — Hue discovers these automatically.

## Configuration Layout

```
homeassistant/configuration/
├── configuration.yaml       # main config
├── automations/             # ~50 YAML files (sl_*, reolink, camera privacy)
├── scripts/                 # ~14 scripts (sl_room_on/off, power_on, etc.)
├── helpers/generated/       # generated by Jinja2 templates
│   ├── input_boolean/
│   ├── input_datetime/
│   ├── input_select/
│   ├── input_number/
│   ├── input_text/
│   └── timer/
├── blueprints/automation/
├── dashboards/
├── packages/                # template sensors (current time window)
└── scenes/                  # UI-managed scenes
```

Regenerate helpers after template edits:
```bash
cd homeassistant/configuration/helpers && bash generate_helpers.sh
```

## Deployment

Managed by **Komodo** — polls `mac-mini-compose` repo every 5 minutes, runs `docker compose up` on changes. Secrets injected by `scripts/inject-secrets.sh` (runs at boot, via Ansible, and as Komodo pre-deploy hook) from BWS.

Config files are **read-only bind-mounted** into the container. UI-created scenes/automations are stored separately in the docker volume.
